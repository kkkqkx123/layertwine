# 项目命名方案（分风格，适配你这套「多Agent+人工混合编辑、分层状态机、自研检查点/分支、Git解耦同步」架构）
贴合核心特质：**分层隔离、多源编辑、状态快照、检查点基线、Agent协同、轻量版本、Git桥接**

## 一、极简技术风（工程首选、适合框架/底层库）
1. **EditLayer** 编辑分层内核，直白贴合分层状态机
2. **StateBasin** 状态盆地，汇聚人工/Agent多源编辑状态
3. **SnapBase** 快照基线，突出不可变快照+自研基线检查点
4. **LayerVault** 分层仓库，分层+版本保管
5. **CheckEdit** 检查点+编辑一体化

## 二、意境隐喻风（高级感、适合正式项目/开源）
1. **Stratum** 地层，对应多层编辑分层、历史沉积检查点
2. **Lamina** 薄层/层理，精准匹配分层架构、轻量隔离
3. **Cordon** 边界隔离，人工/Agent/临时快照严格边界隔离
4. **AetherRepo** 以太仓库，多Agent协同、无形状态流转
5. **Vectis** 基线承载、版本锚定之意

## 三、贴合多Agent协同专属命名
1. **AgentStrata** Agent地层，多Agent分层编辑历史
2. **CollabLayer** 协同分层，人工+Agent混合协同
3. **DualEdit** 双源编辑（人工/Agent）
4. **MultiBasin** 多源状态汇聚基线

## 四、中文命名（接地气、产品/内部系统可用）
1. **层编库** 分层编辑版本库
2. **态枢** 状态中枢，统筹所有编辑状态与检查点
3. **溯编基线** 可溯源编辑基线系统
4. **协层快照** 协同分层快照平台
5. **检点分层** 检查点+分层架构直抒其意

## 五、推荐首选（最适配你整套架构）
### 开源/框架正式名：**Stratum**
寓意：分层沉积、版本历史像地层一样有序留存，完美匹配分层状态机、不可变快照、检查点历史、分支基线所有设计。
### 简洁别名：**StrataEdit**

### 次选首选：**Lamina**
更轻盈、偏技术内核感，适合做底层存储层/状态机库名称。

如果你告诉我是做**开源库、内部工程框架、还是产品系统**，我可以帮你敲定唯一正式名 + 缩写 + 英文域名适配名。
