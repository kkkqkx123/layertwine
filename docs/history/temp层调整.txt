# 最终架构决策：**彻底剥离 TEMP，独立为「不可变快照备份模块」**
你的判断是**架构最优解**：原TEMP层的**职责、生命周期、交互模式、风险诉求**与核心编辑分层/状态机完全割裂，强行纳入分层架构会导致职责混乱、耦合泛滥、状态机臃肿。

**最终定论**：
将原 `temp` 层**完全移出分层架构、移出状态机**，成为一个**独立、自治、解耦的不可变快照备份模块**；核心编辑系统保留纯净的分层状态机，两者仅保留**最小单向安全接口**，无任何耦合。

---

## 一、架构彻底解耦：两大独立模块划分
### 模块1：核心编辑分层状态机（纯净、闭环、无冗余）
**唯一职责**：管理**当前生效的文件编辑状态**，负责人工/Agent修改的隔离、审核、流转、回退、提交，是系统的核心状态机。
- 包含分层：`manual_edit` / `agent_edit` / `approval` / `staged`
- 特性：不可变快照、分区隔离、支持回退/迁移、对接Git提交
- 严格遵循状态机流转规则，**无任何备份/临时验证逻辑**

### 模块2：独立快照备份模块（原TEMP，完全自治）
**唯一职责**：提供**只读备份、离线验证、安全合并**能力，与核心状态机无任何状态共享、无指针耦合、无回退逻辑。
- 形态：独立的轻量快照存储（基于基线diff/完整内容）
- 特性：不可变、可查询、仅单向操作、不影响核心状态
- **不属于任何分层**、**不进入状态机**、**无状态跃迁**

---

## 二、独立快照备份模块：最终铁律设计
### 核心定位
**核心状态的只读备份仓 + 安全验证沙盒**
### 绝对约束（零风险）
1. **完全物理隔离**：独立存储，不共享核心状态机的任何快照、指针、数据；
2. **无状态机属性**：没有分层、没有流转、没有回退、没有迁移；
3. **仅单向操作**：只允许从核心状态机**读取备份**，不写入、不修改、不清空核心状态；
4. **唯一出口**：仅支持将验证后的快照**合并入staged**；
5. **自治生命周期**：自行管理备份的创建、查询、删除，不干扰主流程。

### 仅支持3个安全操作
1. **备份**：从核心状态机（manual/agent_raw/approval）克隆不可变快照到本模块（只读复制，原状态不变）；
2. **查询**：数据库式检索备份快照（按来源、时间、实例ID、分区）；
3. **合并**：将验证通过的快照，安全合并到核心状态机的`staged`层（仅追加，无回退/覆盖）。

---

## 三、纯净版：核心编辑状态机（剥离TEMP后）
### 1. 分层定义（极简、职责单一）
| 分层 | 分区规则 | 核心能力 |
|------|----------|----------|
| manual_edit | 唯一分区 | 人工修改自动归集 |
| agent_edit | 按Agent实例ID分区 | Agent原始修改隔离 |
| approval | 实例ID + Integrated + Unified | 审核、分组合并、汇总 |
| staged | 唯一分区 | 待提交检查点 |

### 2. 纯净状态机流转（闭环、无冗余）
#### 正向流转（唯一主流程）
人工修改 → `manual_edit` → 合并 → `staged`
Agent修改 → `agent_edit` → 审核 → `approval` → 合并 → `staged`
`staged` → 提交检查点 → Git持久化

#### 逆向流转（仅核心状态回退）
`staged` ↔ `manual_edit`/`agent_edit`/`approval`
`agent_edit` ↔ `approval`（Integrated/Unified）

---

## 四、两大模块唯一交互（极简、无耦合）
```
【核心编辑状态机】 ──(只读备份)→ 【独立快照备份模块】
【独立快照备份模块】 ──(安全合并)→ 【核心编辑状态机/staged】
```
- 无反向操作
- 无状态共享
- 无指针关联
- 无互相干扰

---

## 五、终极架构优势（完美解决所有问题）
1. **职责绝对单一**：核心状态机只管「当前生效编辑」，备份模块只管「临时备份验证」，架构极简；
2. **零破坏风险**：备份模块完全隔离，永远无法污染、修改、回退核心编辑状态；
3. **状态机极度纯净**：移除所有无关逻辑，流转规则清晰，无bug、易维护；
4. **可扩展性拉满**：备份模块可独立升级（支持多副本、云端、版本管理），不影响核心系统；
5. **完全符合你的需求**：保留TEMP的备份/验证/合并能力，同时彻底杜绝破坏文件编辑的可能。

---

# 最终总结
1. **TEMP 不再是分层**：彻底退出分层架构，**不属于状态机**；
2. **TEMP 升级为独立模块**：命名为「**不可变快照备份模块**」，自治、隔离、安全；
3. **核心状态机纯净闭环**：仅保留 `manual/agent/approval/staged` 四层，流转规则极简；
4. **架构终态**：核心编辑（状态机）+ 快照备份（工具模块），解耦、安全、可落地。
