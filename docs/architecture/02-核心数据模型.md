# 2. 核心数据模型

## 2.1 实体总览

| 实体类型 | 可变？ | 核心作用 | 归属 |
|----------|--------|----------|------|
| 🔹 **FileNode**（文件基准） | 不可变 | 唯一标识一个文件（路径+哈希） | 基础实体 |
| 🔹 **Delta**（最小增量） | 不可变 | 行级 diff + 元数据 | 最小数据单元 |
| 🔹 **Snapshot**（状态快照） | 不可变 | 文件的完整不可变状态 | 核心状态实体 |
| 🔹 **Partition**（分区） | 可变 | 逻辑容器，持有当前 Snapshot 引用 + 历史 | 层/分组载体 |
| 🔹 **Layer**（分层） | 可变 | 顶级逻辑分组，管理多个 Partition | 顶层分层载体 |

## 2.2 不可变实体

### FileNode（文件唯一标识）

```rust
#[derive(Hash, Eq, PartialEq, Serialize, Deserialize)]
struct FileNode {
    file_path: PathBuf,     // 文件绝对路径（唯一标识）
    base_hash: Blake3Hash,  // 原始文件哈希（不可变基准）
}
```

### Delta（不可变最小修改）

```rust
#[derive(Serialize, Deserialize)]
struct Delta {
    id: DeltaId,            // 唯一 ID（Hash 或 UUID）
    file: FileNode,         // 归属文件
    diff: LineDiff,         // 行级差异
    source: SourceType,     // 来源
    timestamp: i64,         // 时间戳（Unix 毫秒）
}

enum SourceType {
    Manual,                         // 人工编辑
    Agent(AgentInstanceId),         // Agent 实例
    Backup,                         // 备份模块来源
}

struct LineDiff {
    hunks: Vec<Hunk>,       // 多个 diff 块
}

struct Hunk {
    old_start: u32,
    old_lines: u32,
    new_start: u32,
    new_lines: u32,
    content: Vec<DiffOp>,   // Equal / Insert / Delete / Replace
}
```

### Snapshot（不可变状态快照）⭐ 核心实体

```rust
#[derive(Serialize, Deserialize)]
struct Snapshot {
    id: SnapshotId,                 // 唯一 ID（内容哈希）
    file: FileNode,                 // 归属文件
    deltas: Vec<Delta>,             // 该快照包含的所有增量
    parents: Vec<SnapshotId>,       // 父快照引用链
    partition_type: PartitionType,  // 归属分区类型
    created_at: i64,
}
```

**关键设计要点：**

- **单父快照** → 线性修改，支持逐次回退
- **多父快照** → 合并操作，可回退到任意父来源
- 合并不销毁任何历史状态，所有父快照永久存在
- Snapshot 的 `id` 由其内容哈希决定（Content-Addressable）

## 2.3 可变实体

### Partition（分区 — 可变指针）

```rust
struct Partition {
    id: PartitionId,
    name: String,                        // 分区名（如 AGENT_123、INTEGRATED_TASK1）
    current_snapshot: SnapshotId,        // 可变指针：当前生效的快照
    history: Vec<SnapshotId>,            // 历史快照链表（全量保留）
    layer_type: LayerType,               // 归属顶层分层
    created_at: i64,
}
```

### Layer（分层 — 可变容器）

```rust
struct Layer {
    layer_type: LayerType,       // MANUAL / AGENT_RAW / APPROVED / STAGED
    partitions: Vec<Partition>,  // 包含的分区
}
```

## 2.4 分层与分区映射

| 分层 | 对应 Layer | 分区规则 |
|------|-----------|----------|
| manual_edit | `MANUAL` | 唯一分区（不分区） |
| agent_edit | `AGENT_RAW` | 按 Agent 执行实例 ID 分区 |
| approval | `APPROVED` | 按实例 ID 分区 + INTEGRATED-* 分区 + UNIFIED 唯一分区 |
| staged | `STAGED` | 唯一分区（不分区） |

```rust
enum LayerType {
    Manual,
    AgentRaw,
    Approved,
    Staged,
}

enum PartitionType {
    Manual,
    AgentRaw(AgentInstanceId),
    ApprovedAgent(AgentInstanceId),
    Integrated(String),     // 自定义整合分区名
    Unified,
    Staged,
}
```

## 2.5 存储架构

```
┌──────────────────────────────────────────┐
│              SQLite 数据库                  │
│                                            │
│  tables:                                   │
│    file_nodes     (只增,不可变)              │
│    deltas         (只增,不可变)              │
│    snapshots      (只增,不可变)              │
│    partitions     (可更新指针)               │
│    layers         (可更新)                  │
│    checkpoints    (只增,不可变)              │
│    branches       (可更新指针)               │
│    git_anchors    (只增)                    │
└──────────────────────────────────────────┘
```

- 不可变实体：`INSERT ONLY`，永不 `UPDATE` / `DELETE`
- 可变实体：允许 `UPDATE` 指针字段
- GC 机制仅清理不可达的快照/Delta 引用
