## 二、核心不可变数据结构（兼容原有体系，无缝衔接）
基于你之前确定的**不可变快照/Delta**设计，新增3个顶层实体：

### 1. Checkpoint（自研检查点 = 你的提交单元）
**不可变、轻量、替代Git Commit**
```
Checkpoint {
  id: str                # 唯一哈希ID（自研，无需Git）
  parents: list[Checkpoint]  # 父提交（单父=普通提交，多父=分支合并）
  baseline_snapshot: Snapshot  # 对应完整文件基线（核心编辑层的基准）
  metadata: {
    author: str          # 提交人/AgentID
    message: str         # 提交信息
    timestamp: int
    git_anchor: str      # Git同步锚点（绑定的Git Commit哈希，无则为空）
  }
}
```

### 2. Branch（轻量分支 = 仅可变指针）
**无实体数据，仅指向最新Checkpoint**（和Git分支逻辑一致，但极致轻量）
```
Branch {
  name: str              # 分支名（如 task/agent-123、main）
  head: Checkpoint       # 可变指针：指向分支最新检查点
}
```

### 3. CheckpointRepo（仓库总控）
管理所有分支、检查点DAG、当前工作分支
```
Repo {
  branches: list[Branch]
  current_branch: Branch
  checkpoint_dag: DAG    # 所有检查点有向无环图
}
```

## 三、自研检查点式提交（彻底不依赖Git）
### 1. 提交规则（完全对齐你的体系）
- **唯一提交入口**：`staged` 分层（和你最初设计完全一致）
- **提交行为**：
  1. 锁定当前`staged`的不可变快照；
  2. 生成新的`Checkpoint`，父指向当前分支最新检查点；
  3. 移动当前分支`head`指针到新检查点；
  4. 清空`staged`，**核心编辑层基线更新为新检查点**；
- **特性**：轻量（仅存Delta）、不可变、自主管理历史，无Git参与。

### 2. 提交生命周期
`编辑(manual/agent_edit) → 审核(approval) → 合并(staged) → 自研检查点提交 → 分支更新`

## 四、轻量分支管理设计（适配多Agent/多任务）
基于**Checkpoint DAG + 分支指针**实现，无冗余数据：
1. **创建分支**
   基于当前检查点，新建一个指针（零数据复制），用于独立Agent任务/人工编辑；
2. **切换分支**
   自动将**核心编辑层所有分层重置为目标分支的最新基线**（仅切换指针，不修改数据）；
3. **分支合并**
   生成**多父检查点**，保留合并来源（支持Agent分支/人工分支合并），合并冲突可溯源；
4. **隔离性**
   分支间编辑完全隔离，不互相污染。

## 五、Git双向同步设计（对齐基线 + 清理冗余）
### 核心定位
自研仓库 = **实时编辑层**（轻量、分层、分支、快速检查点）
Git = **持久化存储层**（长期归档、远程同步、标准兼容）
**双向同步仅做基线对齐，不干涉你的编辑分层**。

### 1. 同步方向1：Git → 自研仓库（初始化/拉取）
1. 从Git指定`HEAD/分支/提交`导出文件；
2. 在自研仓库生成**初始Checkpoint**，绑定`git_anchor`（Git提交哈希）；
3. 核心编辑层基线重置为该检查点，完成对齐。

### 2. 同步方向2：自研仓库 → Git（推送/清理）
**这是你要的「清理冗余检查点」核心能力**
1. 将当前分支最新Checkpoint（基线）导出为Git提交；
2. 在自研Checkpoint中写入`git_anchor`（绑定Git提交哈希）；
3. **触发冗余检查点GC**：所有**早于该锚点、无Git绑定**的检查点，直接删除。

### 同步铁律
- 同步仅操作**基线**，不同步你的`manual/agent_edit/approval`分层状态；
- 同步后，自研仓库只保留**必要的最新检查点**，Git承担全量历史存储。

## 六、冗余检查点清理（GC机制）
### 清理规则（安全、无数据丢失）
1. **保护规则**
   - 当前分支head检查点 → 保护
   - 绑定`git_anchor`的检查点 → 保护
2. **清理规则**
   - 非当前分支、无Git锚点、超过N天的历史检查点 → 自动删除
   - 同步Git后，所有同步前的冗余检查点 → 批量删除
3. **效果**
   自研仓库永远保持**极小体积**，仅存当前编辑+必要检查点，全量历史托管给Git。
