# 3. 分层状态机设计

## 3.1 状态机总览

核心编辑分层状态机由四层纯净状态构成，每层职责单一，流转规则严格。

```
                     ┌──────────┐
                     │ manual   │ ←── 人工编辑自动归集
                     │ _edit    │
                     └────┬─────┘
                          │ 合并
                          ▼
┌──────────┐  审核  ┌──────────┐  合并  ┌──────────┐  提交  ┌──────────┐
│ agent    │ ─────→ │ approval │ ─────→ │ staged   │ ─────→ │ 检查点   │
│ _edit    │        │          │        │          │        │ 仓库     │
└──────────┘        └──────────┘        └──────────┘        └──────────┘
    ▲                    │  ▲                │
    │                    │  │                │
    └──── 回退 ──────────┘  └── 回退 ────────┘
```

## 3.2 正向流转

### manual_edit → staged

```rust
/// 将 manual_edit 层当前快照合并到 staged
fn merge_manual_to_staged(
    repo: &mut Repo,
) -> Result<SnapshotId> {
    let manual = repo.get_layer(LayerType::Manual);
    let staged = repo.get_layer_mut(LayerType::Staged);
    let manual_snapshot = manual.partitions[0].current_snapshot;
    let staged_snapshot = staged.partitions[0].current_snapshot;

    let new_snapshot = Snapshot::merge(vec![staged_snapshot, manual_snapshot]);
    staged.partitions[0].push_history(new_snapshot);
    Ok(new_snapshot)
}
```

### agent_edit → approval → staged

```rust
/// Agent 修改进入审核层
fn move_agent_to_approval(
    repo: &mut Repo,
    agent_id: AgentInstanceId,
) -> Result<()> {
    let agent_partition = repo.get_partition(LayerType::AgentRaw, &agent_id)?;
    let approval_partition = repo.get_or_create_approval_partition(&agent_id);

    let new_snapshot = Snapshot::merge(vec![
        approval_partition.current_snapshot,
        agent_partition.current_snapshot,
    ]);
    approval_partition.push_history(new_snapshot);
    Ok(())
}

/// 从 Approval 的 Agent 分区迁移到 Integrated 分区
fn move_approval_to_integrated(
    repo: &mut Repo,
    agent_id: AgentInstanceId,
    integrated_name: String,
) -> Result<()> {
    let agent_partition = repo.get_approval_agent_partition(&agent_id)?;
    let integrated = repo.get_or_create_integrated(&integrated_name);

    let new_snapshot = Snapshot::merge(vec![
        integrated.current_snapshot,
        agent_partition.current_snapshot,
    ]);
    integrated.push_history(new_snapshot);
    Ok(())
}

/// 从 Integrated 合并到 Unified
fn merge_integrated_to_unified(
    repo: &mut Repo,
    integrated_names: Vec<String>,
) -> Result<()> {
    let unified = repo.get_unified_partition();
    let mut parents = vec![unified.current_snapshot];

    for name in integrated_names {
        let p = repo.get_integrated(&name)?;
        parents.push(p.current_snapshot);
    }

    let new_snapshot = Snapshot::merge(parents);
    unified.push_history(new_snapshot);
    Ok(())
}
```

### staged → 检查点提交

```rust
fn commit_staged(repo: &mut Repo, message: &str) -> Result<CheckpointId> {
    let staged = repo.get_layer(LayerType::Staged);
    let snapshot = staged.partitions[0].current_snapshot;

    let checkpoint = Checkpoint::new(
        snapshot,
        repo.current_branch.head,
        message,
    );

    repo.checkpoint_dag.add(checkpoint.clone());
    repo.current_branch.head = checkpoint.id;
    // 清空 staged
    repo.reset_layer(LayerType::Staged);
    Ok(checkpoint.id)
}
```

## 3.3 逆向流转（回退）

所有回退操作**仅修改指针**，不修改任何不可变数据。

### 分区自身回退

```rust
fn rollback_partition(partition: &mut Partition) -> Result<()> {
    let prev = partition.history.pop()
        .ok_or(Error::NoHistory)?;
    partition.current_snapshot = prev;
    Ok(())
}
```

### staged → manual / agent_raw

```rust
fn rollback_staged_to_source(
    repo: &mut Repo,
    target_layer: LayerType,
) -> Result<()> {
    let staged = repo.get_partition(LayerType::Staged, &PartitionId::default())?;
    let staged_snapshot = repo.get_snapshot(staged.current_snapshot)?;

    // 从 staged 的 parents 中找到目标层来源
    let target_parent = staged_snapshot.parents
        .iter()
        .find(|pid| repo.get_snapshot(**pid)?.partition_type.belongs_to(target_layer))
        .ok_or(Error::NoValidParent)?;

    staged.current_snapshot = *target_parent;
    Ok(())
}
```

### approval 内部迁移

```rust
// AGENT_RAW ↔ INTEGRATED ↔ UNIFIED 双向迁移（仅切换指针）
fn migrate_between_approval_partitions(
    repo: &mut Repo,
    from: &PartitionId,
    to: &PartitionId,
) -> Result<()> {
    let from_partition = repo.get_partition(LayerType::Approved, from)?;
    let to_partition = repo.get_partition_mut(LayerType::Approved, to)?;
    to_partition.current_snapshot = from_partition.current_snapshot;
    Ok(())
}
```

## 3.4 状态机铁律

1. **不可越层流转**：`agent_edit` 必须经过 `approval` 才能到 `staged`
2. **不可反向写入**：回退仅切换指针，不向源层写入新数据
3. **合并不覆盖**：所有合并操作生成新的 Snapshot，保留所有父状态
4. **原子操作**：每个流转操作要么全成功，要么全失败（事务保证）
