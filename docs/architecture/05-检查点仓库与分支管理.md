# 5. 检查点仓库与分支管理

## 5.1 概述

自研检查点仓库提供**脱离 Git 依赖**的提交历史和分支管理能力。基于不可变快照体系，实现轻量级版本控制。

## 5.2 Checkpoint（自研提交单元）

```rust
#[derive(Serialize, Deserialize)]
struct Checkpoint {
    id: CheckpointId,                       // 唯一哈希 ID（内容寻址）
    parents: Vec<CheckpointId>,             // 父提交（单父=普通，多父=合并）
    baseline_snapshot: SnapshotId,          // 提交时的文件基线
    metadata: CheckpointMetadata,
    created_at: i64,
}

#[derive(Serialize, Deserialize)]
struct CheckpointMetadata {
    author: String,                         // 提交人 / Agent ID
    message: String,                        // 提交信息
    git_anchor: Option<GitCommitHash>,      // Git 同步锚点
}
```

**特性：**
- 不可变、轻量（仅存 Delta，不存全量文件）
- 内容寻址 ID（由内容哈希决定）
- 单父 = 线性提交，多父 = 分支合并
- `git_anchor` 可选绑定 Git 提交

## 5.3 Branch（轻量分支）

```rust
struct Branch {
    name: String,               // 分支名（如 main、task/agent-123）
    head: CheckpointId,         // 可变指针：指向最新检查点
    created_at: i64,
}
```

**分支本质**：仅一个指针，零数据复制。

## 5.4 CheckpointRepo（仓库总控）

```rust
struct CheckpointRepo {
    branches: Vec<Branch>,
    current_branch: usize,              // 当前分支索引
    checkpoint_dag: Dag<CheckpointId>,  // 所有检查点的有向无环图
    layer_state: LayerState,            // 核心编辑层当前状态
}

struct Dag<NodeId> {
    nodes: HashMap<NodeId, HashSet<NodeId>>,  // node → children
    // parents 在 Checkpoint 实体中维护
}
```

## 5.5 分支操作

### 创建分支

```rust
fn create_branch(
    repo: &mut CheckpointRepo,
    name: &str,
    from_checkpoint: Option<CheckpointId>,
) -> Result<()> {
    let head = from_checkpoint
        .unwrap_or(repo.current_branch_head());
    let branch = Branch {
        name: name.to_string(),
        head,
        created_at: chrono::Utc::now(),
    };
    repo.branches.push(branch);
    Ok(())
}
```

### 切换分支

```rust
fn switch_branch(
    repo: &mut CheckpointRepo,
    branch_name: &str,
) -> Result<()> {
    let branch_idx = repo.find_branch(branch_name)?;
    repo.current_branch = branch_idx;

    // 重置核心编辑层基线到目标分支的最新检查点
    let checkpoint = repo.get_checkpoint(repo.current_branch_head())?;
    repo.layer_state.reset_to(checkpoint.baseline_snapshot)?;
    Ok(())
}
```

### 合并分支

```rust
fn merge_branches(
    repo: &mut CheckpointRepo,
    source_branch: &str,
    message: &str,
) -> Result<CheckpointId> {
    let source_head = repo.get_branch_head(source_branch)?;
    let current_head = repo.current_branch_head();

    // 生成多父检查点
    let checkpoint = Checkpoint {
        id: CheckpointId::new(),
        parents: vec![current_head, source_head],
        baseline_snapshot: repo.layer_state.staged_snapshot(),
        metadata: CheckpointMetadata {
            author: "merge".into(),
            message: message.to_string(),
            git_anchor: None,
        },
        created_at: chrono::Utc::now(),
    };

    repo.checkpoint_dag.add_node(checkpoint.id);
    // 子关系
    repo.checkpoint_dag.add_edge(current_head, checkpoint.id);
    repo.checkpoint_dag.add_edge(source_head, checkpoint.id);

    repo.current_branch_mut().head = checkpoint.id;
    Ok(checkpoint.id)
}
```

## 5.6 提交生命周期

```
编辑(manual/agent_edit) → 审核(approval) → 合并(staged)
    → 自研检查点提交 → 分支更新 → 继续编辑
                              ↓
                        可选：Git 同步 → GC 冗余检查点
```

## 5.7 隔离性保障

- 分支间编辑完全隔离，不互相污染
- 切换分支时自动保存当前分支的编辑状态
- 每个分支独立维护 `layer_state`
