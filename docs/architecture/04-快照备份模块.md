# 4. 独立快照备份模块

## 4.1 定位

原 TEMP 层彻底移出分层状态机，成为**独立、自治、解耦的不可变快照备份模块**。

设计目的：存储需要单独在后续阶段验证的快照，与核心编辑层完全物理隔离。

## 4.2 绝对约束

1. **完全物理隔离**：独立存储，不共享核心状态机的任何快照、指针、数据
2. **无状态机属性**：没有分层、没有流转、没有回退、没有迁移
3. **仅单向操作**：只允许从核心状态机**读取备份**，不写入、不修改、不清空核心状态
4. **唯一出口**：仅支持将验证后的快照**合并入 staged**
5. **自治生命周期**：自行管理备份的创建、查询、删除，不干扰主流程

## 4.3 支持的操作

### 备份（从核心状态机克隆快照）

```rust
/// 从核心状态机克隆快照到备份模块
fn backup_snapshot(
    core_repo: &CoreRepo,
    backup_repo: &mut BackupRepo,
    snapshot_id: SnapshotId,
    label: Option<String>,
) -> Result<BackupId> {
    let snapshot = core_repo.get_snapshot(snapshot_id)?;
    let backup = BackupSnapshot {
        id: BackupId::new(),
        source_snapshot: snapshot_id,
        file: snapshot.file.clone(),
        deltas: snapshot.deltas.clone(),
        label,
        backed_at: chrono::Utc::now(),
    };
    backup_repo.store(backup)
}
```

### 查询

```rust
/// 按条件检索备份快照
fn query_backups(
    backup_repo: &BackupRepo,
    filter: BackupFilter,
) -> Result<Vec<BackupSnapshot>> {
    backup_repo.query(filter)
}

struct BackupFilter {
    source: Option<SourceType>,
    time_range: Option<Range<i64>>,
    agent_id: Option<AgentInstanceId>,
    partition: Option<PartitionType>,
    label: Option<String>,
}
```

### 合并入 staged

```rust
/// 将验证通过的备份快照合并到 staged
fn merge_backup_to_staged(
    backup_repo: &BackupRepo,
    core_repo: &mut CoreRepo,
    backup_id: BackupId,
) -> Result<SnapshotId> {
    let backup = backup_repo.get(backup_id)?;
    let staged = core_repo.get_staged_partition_mut();

    // 从备份还原 Snapshot（使用原快照数据）
    let snapshot = core_repo.get_snapshot(backup.source_snapshot)?;
    let merged = Snapshot::merge(vec![staged.current_snapshot, snapshot.id]);
    staged.push_history(merged);
    Ok(merged)
}
```

## 4.4 存储模型

```rust
/// 备份模块的独立存储
struct BackupRepo {
    storage: BackupStorage,
}

struct BackupSnapshot {
    id: BackupId,
    source_snapshot: SnapshotId,       // 来源快照 ID
    file: FileNode,
    deltas: Vec<Delta>,
    label: Option<String>,
    backed_at: DateTime<Utc>,
    metadata: HashMap<String, String>,
}

struct BackupStorage {
    /// 使用独立的 SQLite 数据库文件
    db_path: PathBuf,
}
```

## 4.5 与核心状态机的交互

```
┌──────────────────────┐   备份（只读克隆）   ┌──────────────────────┐
│  核心编辑状态机       │ ─────────────────→  │  独立快照备份模块     │
│                      │                      │                      │
│  manual/agent/       │ ←─────────────────  │  只读查询             │
│  approval/staged     │   合并入 staged       │                      │
└──────────────────────┘                      └──────────────────────┘
```

- 无反向操作
- 无状态共享
- 无指针关联
- 无互相干扰
