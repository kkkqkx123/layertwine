## 目标

实现 manual_edit → agent_edit → approval → staged 四层纯净状态机的完整正向流转和逆向回退，确保分层隔离和不可变原则。

## 任务清单

### 3.1 Layer/Partition 管理层（`state_machine/mod.rs` 和 `core/layer.rs` 扩展）[1.5h]

- [ ] `StateMachine` 结构体 — 持有各 Layer 引用，提供统一操作入口
- [ ] `fn get_partition(layer, partition_id) -> &Partition`
- [ ] `fn get_partition_mut(layer, partition_id) -> &mut Partition`
- [ ] `fn get_or_create_partition(layer, partition_id) -> &mut Partition`
- [ ] 事务性包装：所有流转操作在 SQLite 事务内执行

### 3.2 manual_edit 层（`state_machine/manual.rs`）[1h]

按 [分层状态机 §3.2](../architecture/03-分层状态机.md#32-正向流转) 实现：

- [ ] `fn apply_manual_edit(repo, file, content) -> Result<SnapshotId>`
  - 对比文件 old/new 内容生成 Delta
  - 将 Delta 追加到 manual_edit 层的当前 Snapshot
  - 生成新 Snapshot，更新 partition 指针
- [ ] `fn merge_manual_to_staged(repo) -> Result<SnapshotId>`
  - 取 manual_edit 和 staged 的当前 Snapshot
  - 合并生成新 Snapshot 推入 staged 的历史
  - 参考 [分层状态机 §3.2](../architecture/03-分层状态机.md#32-正向流转) 的伪代码

### 3.3 agent_edit 层（`state_machine/agent.rs`）[1.5h]

按 [分层状态机 §3.2](../architecture/03-分层状态机.md#32-正向流转) 实现：

- [ ] `fn apply_agent_edit(repo, agent_id, file, content) -> Result<SnapshotId>`
  - 每个 Agent 实例隔离在独立 Partition 中
- [ ] `fn move_agent_to_approval(repo, agent_id) -> Result<()>`
  - Agent Raw → Agent Approval Partition 迁移
- [ ] `fn discard_agent_edit(repo, agent_id) -> Result<()>`
  - 放弃 Agent 修改（仅切换指针到父 Snapshot）

### 3.4 approval 层（`state_machine/approval.rs`）[1.5h]

按 [分层状态机 §3.3](../architecture/03-分层状态机.md#33-逆向流转回退) 实现：

- [ ] `fn move_approval_to_integrated(repo, agent_id, integrated_name) -> Result<()>`
  - Agent Approval → Integrated 分区迁移
- [ ] `fn move_integrated_to_unified(repo, integrated_names) -> Result<()>`
  - 多个 Integrated → Unified 合并
- [ ] `fn migrate_between_partitions(repo, from, to) -> Result<()>`
  - AGENT_RAW ↔ INTEGRATED ↔ UNIFIED 双向指针切换

### 3.5 staged 层（`state_machine/staged.rs`）[1h]

- [ ] `fn merge_approval_to_staged(repo, approval_partition) -> Result<SnapshotId>`
  - approval → staged 合并
- [ ] `fn commit_staged_to_checkpoint(repo, message) -> Result<CheckpointId>`
  - staged → Checkpoint 提交（占位，P4 实现具体逻辑）

### 3.6 层间流转协调（`state_machine/transition.rs`）[2h]

按 [分层状态机 §3.4](../architecture/03-分层状态机.md#34-状态机铁律) 的流转规则实现：

- [ ] **正向流转**：
  - `Transition::ManualToStaged` → `merge_manual_to_staged`
  - `Transition::AgentToApproval` → `move_agent_to_approval`
  - `Transition::ApprovalToStaged` → `merge_approval_to_staged`
- [ ] **逆向流转**：
  - `Transition::StagedToManual` → 从 staged 的 parents 找到 manual_edit 来源
  - `Transition::StagedToApproval` → 从 staged 的 parents 找到 approval 来源
  - `Transition::ApprovalToAgentRaw` → approval → agent_raw 回退
- [ ] **铁律检查**：
  - 禁止越层流转（`agent_edit` → `staged` 越过 `approval`）
  - 回退仅切换指针（不反向写入）
  - 合并不覆盖（生成新 Snapshot）

### 3.7 全场景回退实现（`state_machine/transition.rs` 扩展）[1.5h]

按 [设计方案 §九](../设计方案.md) 实现：

- [ ] `fn rollback_partition(repo, partition_id) -> Result<()>`
  - 分区自身回退：`current = history.pop()`
- [ ] `fn rollback_staged_to_layer(repo, target_layer) -> Result<()>`
  - STAGED → MANUAL / AGENT_RAW / APPROVAL 精确回退
- [ ] `fn merge_backup_to_staged(repo, backup_snapshot_id) -> Result<()>`
  - 备份模块 → STAGED 合并（占位，P5 实现）

### 3.8 单元测试 [1.5h]

- [ ] 创建各层的 partition 并验证隔离
- [ ] manual_edit 编辑后合并到 staged
- [ ] agent_edit 改经 approval 到 staged 的全流程
- [ ] 回退到指定层
- [ ] 禁止越层流转（agent_edit → staged 应报错）
- [ ] 事务回滚：失败操作不残留

## 验收标准

- [ ] 全正向流转：manual/agent → approval → staged → Checkpoint
- [ ] 全逆向回退：staged → approval → agent_raw / manual
- [ ] partition 隔离：不同 agent 的修改互不干扰
- [ ] 铁律检查：越层操作被禁止
- [ ] 所有操作事务安全
