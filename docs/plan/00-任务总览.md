# Stratum 分阶段代码实现任务总览

> 本文档基于 [设计方案.md](../设计方案.md) 及 `architecture/`、`reference/` 目录下的详细设计文档，将完整的 Stratum 实现拆分为 8 个可独立交付的阶段。

## 阶段总览

| 阶段 | 名称 | 里程碑 | 预估工作量 | 前置依赖 |
|------|------|--------|-----------|---------|
| **P1** | 核心数据模型与 SQLite 存储层 | 所有不可变实体定义 + 建表 + 增删查 | 2-3 天 | 无 |
| **P2** | Snapshot Diff/Merge 引擎 | `similar` 集成 + Delta 生成 + Merge + inverse | 2-3 天 | P1 |
| **P3** | 分层状态机（四层流转） | manual→agent→approval→staged 全流转 | 3-4 天 | P1, P2 |
| **P4** | 检查点仓库与分支管理 | Checkpoint 提交 + Branch + DAG | 2-3 天 | P1, P3 |
| **P5** | 独立快照备份模块 | 备份/查询/合并入 staged | 1-2 天 | P1, P3 |
| **P6** | Git 双向同步与 GC | `init_from_git` + `push_to_git` + 冗余清理 | 2-3 天 | P4 |
| **P7** | CLI 命令行接口 | `clap` 子命令 + 格式化输出 + 错误处理 | 2-3 天 | P3~P6 |
| **P8** | 测试覆盖与边界加固 | 单元测试 + 集成测试 + 模糊测试 | 2-3 天 | P1~P7 |
| **P9** | 存储层重构与接口加固 | 依赖倒置 + 事务一致性 + 技术债务清理 | 2-3 天 | P1~P8 |

**总预估**：约 16-25 天（单人开发）

## 依赖关系图

```
P1 ──→ P2 ──→ P3 ──→ P4 ──→ P6 ──→ P7

```

## 项目结构（终态）

参考 [架构总览](../architecture/01-架构总览.md) 和 [Rust实现方案](../architecture/07-Rust实现方案.md) 中的项目结构，终态代码结构如下：

```
stratum/
├── Cargo.toml
├── src/
│   ├── main.rs              # CLI 入口（P7）
│   ├── lib.rs                # 库入口
│   ├── core/                 # 核心数据类型（P1）
│   │   ├── file_node.rs
│   │   ├── delta.rs
│   │   ├── snapshot.rs
│   │   ├── partition.rs
│   │   ├── layer.rs
│   │   └── types.rs
│   ├── engine/               # Diff/Merge 引擎（P2）
│   │   ├── diff.rs
│   │   ├── merge.rs
│   │   └── inverse.rs
│   ├── state_machine/        # 分层状态机（P3）
│   │   ├── manual.rs
│   │   ├── agent.rs
│   │   ├── approval.rs
│   │   ├── staged.rs
│   │   └── transition.rs
│   ├── backup/               # 快照备份模块（P5）
│   │   ├── backup_repo.rs
│   │   └── backup_snapshot.rs
│   ├── checkpoint/           # 检查点仓库（P4）
│   │   ├── checkpoint.rs
│   │   ├── branch.rs
│   │   ├── repo.rs
│   │   └── dag.rs
│   ├── git_sync/             # Git 同步（P6）
│   │   ├── git_bridge.rs
│   │   └── gc.rs
│   ├── storage/              # 持久化层（P1）
│   │   ├── sqlite_storage.rs
│   │   ├── migrations.rs
│   │   └── repository.rs
│   ├── cli/                  # CLI 接口（P7）
│   │   ├── commands.rs
│   │   └── output.rs
│   └── error.rs              # 全局错误类型（P1）
```
