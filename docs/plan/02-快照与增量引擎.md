## 目标

基于 `similar` crate 实现行级 diff 生成、Snapshot 构造与合并、逆向 delta 生成，为状态机流转提供核心运算能力。

## 任务清单

### 2.1 添加依赖 [0.1h]

- [ ] `cargo add similar`（默认特性即可：`std` + `text`）

### 2.2 Diff 引擎（`engine/diff.rs`）[2h]

参考 [similar-diff引擎 §1.4](../reference/01-similar-diff引擎.md) 和 [复用分析 §1](../reference/05-复用分析.md#1-similar--直接依赖cargotoml)：

- [ ] `fn diff_to_line_diff(old: &str, new: &str) -> LineDiff`
  - 使用 `TextDiff::from_lines` 生成行级 diff
  - 分组 `grouped_ops(3)` 生成 Hunk 列表
  - 将 `DiffOp` 转换为内部 `Hunk` + `DiffOp` 表示
- [ ] `fn diff_to_delta(file: &FileNode, old_content: &str, new_content: &str, source: SourceType) -> Delta`
  - 调用 `diff_to_line_diff` 生成差异
  - 包装为 Delta 实体（包含文件归属、来源、时间戳）
- [ ] 大文件 deadline 支持（参考 [similar-diff引擎 §1.5](../reference/01-similar-diff引擎.md#15-关键技术细节)）
  - `TextDiff::configure().timeout(Duration::from_secs(1))`

### 2.3 Snapshot 构造（`core/snapshot.rs` 扩展）[1.5h]

- [ ] `SnapshotBuilder` 模式（参考 [jj 架构 §2.1.3](../reference/02-jj-jujutsu架构.md#213-commitbuilder-模式)）
  ```rust
  let snapshot = SnapshotBuilder::new(file_node)
      .add_delta(delta_id)
      .with_parent(parent_snapshot_id)
      .with_partition_type(PartitionType::Manual)
      .build()?;
  ```
- [ ] `Snapshot::new_initial(file_node, delta) -> Snapshot` — 从原始文件构建初始快照
- [ ] `Snapshot::apply_delta(snapshot, delta) -> Snapshot` — 在现有快照上应用增量生成新快照

### 2.4 三路合并（`engine/merge.rs`）[2h]

- [ ] `fn merge_snapshots(base: &Snapshot, ours: &Snapshot, theirs: &Snapshot) -> Result<Snapshot>`
  - 使用 `similar` 的三路 diff 能力
  - 行级冲突检测（基于 `DiffOp` 的冲突区域）
  - 无冲突时直接合并
  - 冲突标记（类似 Git 的 `<<<<<<<` / `=======` / `>>>>>>>` 格式）
- [ ] `fn apply_deltas(base_content: &str, deltas: &[Delta]) -> Result<String>`
  - 依次应用 Delta 到文件内容上
  - 支持增量链式应用

### 2.5 逆向 Delta（`engine/inverse.rs`）[1h]

参考 [Immer §4.3](../reference/04-immer不可变数据.md#43-patches-系统) 的 inversePatches 设计：

- [ ] `fn inverse_delta(delta: &Delta) -> Delta`
  - `Insert` ↔ `Delete` 互换
  - `Replace` 内容互换
- [ ] `fn inverse_snapshot(snapshot: &Snapshot) -> Result<Vec<Delta>>`
  - 生成可恢复前一个状态的一系列逆向 Delta

### 2.6 单元测试 [1h]

- [ ] 测试 `diff_to_line_diff`：相同输入→空、插入行、删除行、修改行、多行变更
- [ ] 测试 `apply_deltas`：空白+增量→完整内容、链式应用
- [ ] 测试 `merge_snapshots`：无冲突合入、有冲突标记
- [ ] 测试 `inverse_delta`：delta + inverse → 零效果
- [ ] 测试大文件 deadline 不 panic

## 验收标准

- [ ] `cargo test` 通过
- [ ] 两个文本文件可以计算 LineDiff
- [ ] Delta 可应用到 Snapshot 生成新状态
- [ ] 三路合并在无冲突时自动合并，有冲突时生成冲突标记
- [ ] Delta 与其 inverse 组合应用后回到原始状态
