# P4 — 检查点仓库与分支管理

> 参考文档：
> - [检查点仓库与分支管理](../architecture/05-检查点仓库与分支管理.md) — Checkpoint/Branch/DAG 设计
> - [Rust实现方案 §7.6](../architecture/07-Rust实现方案.md) — SQLite checkpoints/branches 表

> - [Git-delta存储 §3.4](../reference/03-git-delta存储.md) — Commit-Graph 格式参考

## 目标

实现自研检查点仓库：Checkpoint 提交流程、轻量分支创建/切换/合并、DAG 历史追踪。这是独立于 Git 的版本管理核心。

## 任务清单

### 4.1 Checkpoint 实体（`checkpoint/checkpoint.rs`）[1h]

按 [检查点仓库 §5.2](../architecture/05-检查点仓库与分支管理.md) 实现：

- [ ] `Checkpoint` 结构体 — `id`, `parents`, `baseline_snapshot`, `metadata`, `created_at`
- [ ] `CheckpointMetadata` — `author`, `message`, `git_anchor`
- [ ] `CheckpointBuilder` — 链式构造（参考 jj CommitBuilder）
- [ ] 内容寻址 ID 计算（由内容哈希决定）

### 4.2 Branch 实体（`checkpoint/branch.rs`）[0.5h]

按 [检查点仓库 §5.3](../architecture/05-检查点仓库与分支管理.md) 实现：

- [ ] `Branch` 结构体 — `name`, `head: CheckpointId`, `created_at`
- [ ] `fn new(name) -> Self`
- [ ] `fn set_head(checkpoint_id) -> ()`

### 4.3 DAG 图结构（`checkpoint/dag.rs`）[1.5h]

按 [检查点仓库 §5.4](../architecture/05-检查点仓库与分支管理.md) 和 [Git-delta存储 §3.4](../reference/03-git-delta存储.md) 实现：

- [ ] `CheckpointDag` — `nodes: HashMap<CheckpointId, HashSet<CheckpointId>>` (node → children)
- [ ] `fn add_node(id)` — 添加节点
- [ ] `fn add_edge(parent, child)` — 添加父子关系
- [ ] `fn ancestors(id) -> Vec<CheckpointId>` — 获取所有祖先（线性遍历）
- [ ] `fn is_ancestor(ancestor, descendant) -> bool` — 可达性判断
- [ ] `fn merge_base(id1, id2) -> Option<CheckpointId>` — 寻找共同祖先
- [ ] Generation number 维护（用于快速可达性判断）
- [ ] 反向索引：parent → children 的查询

### 4.4 CheckpointRepo 仓库（`checkpoint/repo.rs`）[2h]

按 [检查点仓库 §5.5](../architecture/05-检查点仓库与分支管理.md) 实现：

- [ ] `CheckpointRepo` 结构体 — `branches`, `current_branch`, `checkpoint_dag`, 存储引用
- [ ] **提交**：`fn commit(repo, message) -> Result<CheckpointId>`
  - 将 staged 层的当前状态打包为 Checkpoint
  - 添加到 DAG，更新分支 head
  - 参考 [检查点仓库 §5.5](../architecture/05-检查点仓库与分支管理.md) 的 `commit_staged` 伪代码
- [ ] **分支操作**：
  - `fn create_branch(repo, name) -> Result<()>` — 基于当前 head 新建分支
  - `fn switch_branch(repo, name) -> Result<()>` — 切换分支，保存当前编辑状态
  - `fn list_branches(repo) -> Vec<Branch>`
- [ ] **合并**：`fn merge_branches(repo, source_branch, message) -> Result<CheckpointId>`
  - 生成多父 Checkpoint
  - 参考 [检查点仓库 §5.5](../architecture/05-检查点仓库与分支管理.md) 的 `merge_branches` 伪代码
- [ ] **日志**：`fn log(repo, count) -> Vec<Checkpoint>`
  - 从当前 head 回溯祖先链

### 4.5 SQLite 持久化（`storage/sqlite_storage.rs` 扩展）[1h]

- [ ] 建表：`checkpoints`, `branches`（参考 [Rust实现方案 §7.6](../architecture/07-Rust实现方案.md) 的 SQL）

- [ ] 实现 Branch 的 CRUD

### 4.6 隔离性保障 [1h]

按 [检查点仓库 §5.7](../architecture/05-检查点仓库与分支管理.md) 实现：

- [ ] 切换分支时自动保存当前分支的编辑状态（layer_state 快照）
- [ ] 切换分支后所有分层基线重置为目标分支最新 Checkpoint
- [ ] 分支间编辑完全隔离

### 4.7 单元测试 [1h]

- [ ] 创建 Checkpoint 并验证 ID 内容寻址

## 验收标准

- [ ] staged → Checkpoint 提交成功
- [ ] 分支创建/切换/合并全流程可用
- [ ] DAG 历史可查询（`log` 命令）
- [ ] 分切换后编辑状态正确隔离
- [ ] 合并操作生成多父 Checkpoint
