# E2E 测试使用指南

## 目录结构

```
tests/
├── common/              # 公共测试辅助模块
│   ├── mod.rs          # 模块入口
│   ├── fixture.rs      # 测试环境和 Fixture
│   ├── helpers.rs      # 测试辅助函数
│   ├── output.rs       # 格式化输出工具
│   └── assertions.rs   # 自定义断言
├── e2e/                # 端到端测试
│   ├── mod.rs          # E2E 模块入口
│   ├── basic_workflow.rs      # 基础工作流测试
│   ├── multi_agent.rs         # 多 Agent 协作测试
│   ├── branch_operations.rs   # 分支管理测试
│   ├── backup_restore.rs      # 备份恢复测试
│   └── approval_flow.rs       # 审批流程测试
└── existing_integration/      # 现有集成测试（保留）
    ├── storage_integration.rs
    ├── engine_integration.rs
    └── ...
```

## 运行测试

### 运行所有测试
```bash
cargo test
```

### 只运行 E2E 测试
```bash
cargo test --test e2e
```

### 运行特定测试文件
```bash
cargo test --test e2e::basic_workflow
cargo test --test e2e::multi_agent
cargo test --test e2e::branch_operations
cargo test --test e2e::backup_restore
cargo test --test e2e::approval_flow
```

### 运行单个测试用例
```bash
cargo test test_complete_edit_workflow
cargo test test_multi_agent_collaboration
```

### 显示测试输出
```bash
cargo test --test e2e -- --nocapture
cargo test test_complete_edit_workflow -- --nocapture
```

## 公共模块使用

### TestEnvironment

创建测试环境：

```rust
use crate::common::fixture::{TestEnvironment, TestConfig};

// 创建默认测试环境
let env = TestEnvironment::new(TestConfig::default());

// 创建带 Git 仓库的测试环境
let env = TestEnvironment::with_git(TestConfig::default());
```

### 辅助函数

```rust
use crate::common::helpers::*;

// 初始化仓库
init_repository(&env);

// 应用编辑
let snapshot_id = apply_edit(&env, "test.txt", "content");

// 应用 Agent 编辑
let snapshot_id = apply_agent_edit(&env, "agent-1", "test.txt", "content");

// 提交 Agent
submit_agent(&env, "agent-1");

// 批准 Agent
approve_agent(&env, "agent-1", "feature-name");

// 提交更改
commit_changes(&env, "Commit message", "author");

// 获取日志
let log = get_log(&env, Some(10));

// 获取状态
let status = get_status(&env);
```

### 输出工具

```rust
use crate::common::output::*;

// 打印测试头部
print_test_header("test_name");

// 打印信息
print_info("Information message");
print_success("Success message");
print_error("Error message");
print_warning("Warning message");

// 打印分隔符
print_separator();

// 打印层级状态
print_layer_state_from_status(&status);
print_all_layer_states(&all_partitions);

// 打印文件内容
print_file_content(&content, 10);

// 打印测试结果
print_test_result(true, "test_name", Some(duration));
```

### 自定义断言

```rust
use crate::common::assertions::*;

// 断言快照 ID 有效
assert_valid_snapshot_id(&snapshot_id);

// 断言分区存在
assert_partition_exists(&env, "partition-name");

// 断言分区不存在
assert_partition_not_exists(&env, "partition-name");

// 断言层级分区数量
assert_layer_partition_count(&env, LayerType::ManualEdit, 1);

// 断言日志条目数量
assert_log_entry_count(&env, 2);

// 断言日志包含消息
assert_log_contains(&env, "Commit message");

// 断言文件内容
assert_file_content(&env, &snapshot_id, "expected content");

// 断言快照不同
assert_snapshots_different(&id1, &id2);

// 断言快照相同
assert_snapshots_equal(&id1, &id2);
```

## 编写新测试

### 测试模板

```rust
use layertwine::api::{InitRequest, CommitRequest};
use crate::common::fixture::{TestEnvironment, TestConfig};
use crate::common::helpers::*;
use crate::common::output::*;
use crate::common::assertions::*;

#[test]
fn test_new_feature() {
    let config = TestConfig::default();
    let env = TestEnvironment::new(config.clone());

    print_test_header("test_new_feature");

    print_info("Step 1: Setup");
    init_repository(&env);
    print_success("Setup completed");

    print_info("Step 2: Execute feature");
    // ... feature implementation ...

    print_info("Step 3: Verify results");
    // ... verification ...

    print_test_result(true, "test_new_feature", None);
}
```

### 最佳实践

1. **清晰的测试步骤**：每个测试都分为清晰的步骤，每个步骤都有说明
2. **详细的输出**：使用 `print_*` 函数提供详细的执行信息
3. **完整的验证**：每个步骤完成后都进行验证
4. **状态检查**：使用辅助函数检查系统状态
5. **断言明确**：使用自定义断言提供明确的错误信息
6. **可维护性**：提取公共逻辑到辅助函数

## 故障排查

### 测试失败时

1. 查看完整的测试输出：`cargo test --test e2e -- --nocapture`
2. 检查数据库路径和权限
3. 验证 Git 仓库配置（如果使用）
4. 检查临时目录是否正确清理

### 常见问题

**问题**：测试时数据库文件被锁定
**解决**：确保每个测试使用独立的临时目录

**问题**：Git 仓库初始化失败
**解决**：检查系统是否安装了 Git 并配置了用户信息

**问题**：输出不完整
**解决**：使用 `--nocapture` 参数查看完整输出

## 扩展测试

### 添加新的测试文件

1. 在 `tests/e2e/` 目录创建新的 `.rs` 文件
2. 在 `tests/e2e/mod.rs` 中添加模块声明
3. 遵循现有的测试结构和命名约定

### 添加新的测试场景

1. 分析新场景的测试需求
2. 设计测试步骤和验证点
3. 使用现有的辅助函数和输出工具
4. 确保测试的独立性和可重复性
