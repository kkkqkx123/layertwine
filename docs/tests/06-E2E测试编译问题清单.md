# E2E 测试编译问题清单

## 问题分类

### 1. 模块文件找不到
- **错误信息**: `error[E0583]: file not found for module 'basic_workflow'`
- **影响文件**: tests/e2e_tests.rs
- **原因**: 模块文件名不匹配，Rust 要求模块文件名必须与模块声明名称一致
- **修复方案**: 将 tests/e2e_basic_workflow.rs 重命名为 tests/basic_workflow.rs 等

### 2. 类型导入错误
- **错误信息**: `error[E0432]: unresolved imports 'layertwine::core::FileNode', 'layertwine::core::Snapshot', 'layertwine::core::Partition'`
- **影响文件**: tests/common/fixture.rs
- **原因**: `layertwine::core` 模块没有直接导出 `Partition` 类型
- **修复方案**: 从正确的模块导入类型
  - `FileNode` -> `layertwine::core::file_node::FileNode`
  - `Snapshot` -> `layertwine::core::snapshot::Snapshot`
  - `Partition` -> `layertwine::core::partition::Partition`

### 3. DiffOp 字段名错误
- **错误信息**: `error[E0559]: variant 'DiffOp::Insert' has no field named 'line_num'`
- **影响文件**: tests/common/fixture.rs
- **原因**: `DiffOp` 枚举的字段名与我假设的不同
- **实际字段**:
  - `DiffOp::Insert { new_start, lines }`
  - `DiffOp::Delete { old_start, count }`
  - `DiffOp::Replace { old_start, old_count, new_start, lines }`
- **修复方案**: 修改 fixture.rs 中的 DiffOp 创建代码，使用正确的字段名

### 4. 方法不存在
- **错误信息**: `error[E0599]: no method named 'get_snapshot' found for struct 'SqliteStorage'`
- **影响文件**: tests/common/helpers.rs
- **原因**: `SqliteStorage` 没有 `get_snapshot` 方法
- **修复方案**: 检查可用的方法，可能需要使用其他 API

### 5. 结构体字段访问错误
- **错误信息**: `error[E0609]: no field 'stats' on type '&DiffStats'`
- **影响文件**: tests/common/output.rs
- **原因**: `DiffStats` 的字段结构不正确
- **修复方案**: 检查 `DiffStats` 的实际定义，修复字段访问

### 6. 类型别名使用错误
- **错误信息**: `error[E0423]: expected function, tuple struct or tuple variant, found type alias 'SnapshotId'`
- **影响文件**: tests/common/assertions.rs
- **原因**: `SnapshotId` 是 `ContentId` 的类型别名，而 `ContentId` 是元组结构体
- **修复方案**: 使用 `ContentId([1u8; 32])` 而不是 `SnapshotId([1u8; 32])`

### 7. 所有权问题
- **错误信息**: `error[E0382]: borrow of moved value: 'layer_type'`
- **影响文件**: tests/common/assertions.rs
- **原因**: `LayerType` 没有实现 `Copy` trait，导致所有权移动
- **修复方案**: 克隆值或修改函数签名使用引用

### 8. 模块引用错误
- **错误信息**: 在测试文件中使用 `crate::common`
- **影响文件**: tests/e2e_*.rs
- **原因**: 在测试文件中，`crate` 指向测试二进制文件根，而不是项目根
- **修复方案**: 使用相对路径导入 `common` 模块
