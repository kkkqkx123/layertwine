## 主要问题分析

### 1. 日志数量不匹配
**问题描述**: 测试期望日志有特定数量的条目，但实际包含额外的 root checkpoint

**示例错误**:
```
assertion `left == right` failed: Log should have 1 entries, but has 2
  left: 2
 right: 1
```

**原因**: 系统在初始化时会自动创建一个 root checkpoint，导致日志条目数比预期多1

**影响**: 所有依赖日志数量的测试都受到影响

### 2. 分区名称不匹配
**问题描述**: 期望的分区名称与实际的分区名称不同

**实际分区名称**:
- `manual_edit` 而不是 `manual`
- `staged` 保持不变

**影响**: 分区查找和验证失败

### 3. API 行为差异
**问题描述**: 测试假设的 API 行为与实际行为不一致

**具体问题**:
- 编辑操作可能不会立即创建快照
- 提交操作可能涉及额外的内部步骤
- 分区创建的时机可能与预期不同

### 4. 状态转换逻辑
**问题描述**: 6层 pipeline 的状态转换可能比测试假设的更复杂

**实际流程**:
- 某些操作可能涉及多个层的更新
- 状态可能是异步更新的
- 层级间的依赖关系可能更复杂
