# Stratum 项目性能基准测试分析

## 概述

本文档总结了 Stratum 项目的性能基准测试实现，包括测试覆盖范围、测试方法和预期性能目标。

## 测试覆盖的核心功能

### 1. Diff 引擎 (`diff.rs`)

**测试内容：**
- 行级差异计算性能
- 统一diff格式化性能
- 不同文件大小和变更率的影响

**测试场景：**
- 文件大小：10行、100行、1000行、10000行
- 变更率：10%、30%、50%
- 操作类型：diff_to_line_diff、format_unified_diff

**性能关键点：**
- similar::TextDiff 的算法效率
- Hunk 分组和上下文处理
- 文本处理和字符串操作

### 2. Merge 引擎 (`merge.rs`)

**测试内容：**
- Delta链应用性能
- 三路文本合并性能
- 多个Delta顺序应用

**测试场景：**
- Delta数量：1、5、10个
- 文件大小：10行到1000行
- 操作类型：apply_deltas、merge_texts

**性能关键点：**
- Hunk 排序和位置计算
- 冲突检测算法
- 内容重建的内存分配

### 3. Inverse 引擎 (`inverse.rs`)

**测试内容：**
- 反向Delta生成性能
- 不同操作类型的反向性能
- old_content 对性能的影响

**测试场景：**
- 文件大小：10行、100行、1000行
- 变更率：10%、30%、50%
- 操作类型：Insert、Delete、Replace
- 有/无old_content

**性能关键点：**
- 反向操作的算法复杂度
- 内存访问模式
- 旧内容的查找效率

### 4. 核心类型计算 (`core_types.rs`)

**测试内容：**
- ContentId 计算（Blake3哈希）
- Delta ID 计算
- Snapshot ID 计算
- ID 序列化/反序列化

**测试场景：**
- 数据大小：16字节到16KB
- 文件大小：10行到1000行
- 操作类型：ID计算、十六进制转换

**性能关键点：**
- Blake3 哈希算法速度
- serde_json 序列化效率
- 内存分配和拷贝

### 5. 存储操作 (`storage.rs`)

**测试内容：**
- Snapshot 存储/查询性能
- Delta 存储/查询性能
- 批量操作性能
- FileNode 存储性能

**测试场景：**
- 文件大小：10行、100行、1000行
- 批量大小：10、50个
- 操作类型：store、get、batch_store

**性能关键点：**
- SQLite 写入性能
- WAL 模式的效果
- 索引查询效率
- 事务处理开销

### 6. 序列化/反序列化 (`serialization.rs`)

**测试内容：**
- Delta 序列化/反序列化
- Snapshot 序列化/反序列化
- LineDiff 序列化
- FileNode 序列化

**测试场景：**
- 文件大小：10行到1000行
- 变更率：10%、30%、50%
- 复杂度：不同数量的Hunk

**性能关键点：**
- serde_json 性能
- 二进制 vs JSON 序列化
- 内存使用效率

## 测试方法

### 基准测试框架
- 使用 Criterion 库
- 自动多次迭代采样
- 统计分析和异常值检测
- HTML 报告生成

### 测试数据生成
- 伪随机文本生成
- 可控的变更率
- 不同规模的测试数据
- 真实场景模拟

### 测试覆盖策略
- 大小覆盖：小型（<100行）、中型（100-1000行）、大型（>1000行）
- 操作覆盖：所有核心操作类型
- 边界情况：空文件、大变更率等

## 性能优化建议

### 1. Diff 引擎优化
- 考虑使用更高效的字符串处理库
- 优化 Hunk 分组算法
- 减少不必要的字符串拷贝

### 2. Merge 引擎优化
- 优化 Delta 应用顺序
- 预分配结果缓冲区
- 考虑并行化处理

### 3. 存储优化
- 使用批量插入减少事务开销
- 优化 SQLite 配置参数
- 考虑使用连接池

### 4. 序列化优化
- 考虑使用二进制序列化格式
- 优化序列化缓冲区管理
- 减少不必要的字段序列化

## 性能基准

基于初步测试，以下是建议的性能基准：

### Diff 引擎
- 小型文件（<100行）：<1μs
- 中型文件（100-1000行）：<10μs
- 大型文件（1000-10000行）：<100μs

### Merge 引擎
- 单个Delta应用：<1μs
- 三路合并（小型）：<5μs
- 三路合并（大型）：<50μs

### 内容哈希
- Blake3 哈希：~1GB/s
- ID 计算（100行文件）：<1μs

### 存储操作
- Snapshot 存储：<1ms（100行）
- Delta 存储：<1ms（100行）
- 查询操作：<100μs

### 序列化
- Delta 序列化：<10μs（100行）
- Snapshot 序列化：<10μs
- JSON 解析：~100MB/s
